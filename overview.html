Overview: Native Android Development Using Terminal-IDE


Modern tablets are a far cry from the tiny little CPUs from back in the days of Palm Pilots and such. I thought that it would be nice to develop code directly on the device itself, rather than using an emulator and shuffling code back and forth (to say nothing of having to carry around a full size laptop to do the work on).


Some Android devices, such as the Nexus 7, come with a dual purpose USB port ("On The Go" - OTG) that lets the tablet act as the computer, as well as as a peripheral. This allows you to hook up a keyboard, mouse, or a game controller to the tablet. Almost all support bluetooth, so you can attach an external keyboard, or mouse, that way as well.


Great, now all I need is an editor and some kind of compiler.


While there is a graphical IDE available for Android, it is not free. In addition, I'm a bit of an old school unix guy, so when I stumbled across this app, “Terminal-IDE”, I knew I had to try it.


The Terminal-IDE package includes a few shell tools that are not in the standard Android setup. See the …/system/bin for what is installed. While it has tools to support creation of command line interface (“CLI”) style C and Java programs, I am going to focus on using it for “native” Android applications.


In the application's installation, you will find a directory called …/system/src/demo_android/, in which are a number of scripts and source code. However, for my demo, I chose to make a slightly longer Java/Android program, and to use “make” instead of separate scripts.


By default, I have the Makefile set up to build a signed Android package, and stop. However, there are also "install" and "run" targets as well. The Android installer will prompt you to run the app if the install is successful, so "make install" will do everything you need to compile, install and run.

Runing the build and install process as described above, you will see the following run:

  • aapt - generate binary version of resources and a source code file with reference IDs.
  • javac - a Dalvik port of the Java compiler to generate Java .class files.
  • dx - convert .class and .jar files to .dex format.
  • apkbuilder - generate an (unsigned) Android package, .apk, from the .dex and resource files.
  • signer - generate signed .apk from unsigned .apk.
  • am - install and run the signed .apk copied to the "SD card".
